# 命镜 MingMirror · 产品需求文档（PRD）

> **版本**：v0.1
> **日期**：2026-07-01
> **定位**：不是传统命理网站，而是「个人命运的数字孪生与 AI 决策副驾驶」。

---

## 1. 一句话产品

**命镜（MingMirror）= 你的出生信息 × 多学派命理 Agent × 时间序列推演 × 现实事件反馈，生成一张会进化的「人生导航图」。**

我们不做「输入八字 → 输出一段泛泛批文」的传统命理网页。我们要让用户看见：**此刻的运势天气、不同选择的可能分支、一群 AI 命理师在圆桌前辩论、以及针对你真实目标的择日行动方案。**

---

## 2. 核心理念：从「算命」到「命运操作系统」

传统命理平台的本质是**内容消费**：用户付费买一段解释。

命镜的本质是**决策基础设施**：

- 把命盘当成一个**动态数据库**，而不是静态文本。
- 把「大运/流年/流月/流日」当成**时间序列信号**。
- 把不同命理学派当成**特征模型**，用 ensemble 融合。
- 把用户真实人生事件当成**反馈数据**，持续校准预测。
- 把「择日」从黄历查询升级成**个人目标的最优时机求解器**。

最终目标：让命镜成为用户做重大人生决策时，愿意打开的第一个辅助工具。

## 3. 十大差异化创新点

| 创新点 | 传统平台 | 命镜 |
|--------|---------|------|
| **1. 数字孪生仪表盘** | 静态报告 | 实时「运势天气」+ 人生阶段雷达图 |
| **2. 多 Agent 命理议会** | 单一模型输出 | 八字/紫微/七政/盲派/心理学 Agent 可视辩论 |
| **3. 事件反推校准** | 不收集反馈 | 用户录入重大事件 → 反推时辰 → 修正模型 |

| **5. 命运沙盒 A/B** | 无 | 模拟「换时辰/换城市/换选择」的人生差异 |
| **6. 命运剧本** | 文言文批文 | 以 RPG 角色卡 + 人生章节叙事呈现 |
| **7. 联邦知识图谱** | 大师经验封闭 | 匿名化众包案例 → 持续训练预测模型 |
| **8. 隐私优先架构** | 数据上云 | 本地排盘 + 差分隐私贡献知识 |

## 4. 目标用户

### 核心用户
- **25–40 岁都市人群**：面临职业、婚恋、置业、健康决策，对传统文化有兴趣但不迷信。
- **二次使用者**：用过传统命理 App，感觉「说得像又不像」，渴望更个性化、可验证的体验。

### 高价值用户
- **命理师/咨询师**：用命镜做第二大脑，快速生成报告、管理客户、验证案例。
- **企业 HR / 创始人**：用于团队角色匹配、重大会议择日、品牌战略时机（toB 增值模块）。

###  NOT 目标用户
- 寻求「改运玄学产品」的迷信用户（我们不卖符咒、法事、开光物品）。
- 完全排斥传统文化的纯理性主义者。

## 5. 核心产品模块

### 模块 1：命镜数字孪生（Life Twin Dashboard）

**产品形态**：打开 App 第一眼看到的主页，类似「人生天气」+「角色状态面板」。

**功能细节**：

- **今日运势卡**：用天气隐喻（晴/多云/雨/雷暴）+ 五行能量条 + 建议行动标签。
- **人生阶段雷达图**：事业、财富、婚姻、健康、学业、人际六个维度的「当前能量值」与「未来 12 个月趋势」。
- **大运时间轴**：可滑动查看每十年大运，点击显示该阶段关键词与历史/未来事件标注。
- **能量日历**：把每一天按「宜/忌/平」着色，支持切换查看「事业日」「感情日」「投资日」。

**创新点**：不是给用户一段文字，而是给用户一张**可交互的人生状态图**。

---

### 模块 2：命理议会（Council of Seers）

**产品形态**：一个圆桌会议界面，每个 Agent 有头像、学派身份、发言气泡。

**Agent 角色**：

1. **子平八字师**：格局、旺衰、用神忌神。
2. **盲派做功师**：宾主、体用、做功效率。
3. **紫微斗数师**：十二宫、主星、四化、大限流年。
4. **七政四余师**：日月光照、四余吉凶、命宫身宫。
5. **现代心理师**：把命理结论转译成可执行的心理/行为建议。
6. **数据校准员**：基于用户历史事件，评估各 Agent 预测与该用户的匹配度。

**交互方式**：

- 用户提问后，所有 Agent 同步给出结论。
- 系统高亮「一致结论」与「分歧点」。
- 用户可点击任意 Agent 头像追问：「你为什么和紫微结论不一样？」
- 最终输出「议会决议」+「 dissent 保留意见」。

**创新点**：把黑盒 LLM 输出变成**可审计、可辩论、可追问的多智能体会议**。

---

### 模块 3：事件校准引擎（Event Calibration）

**产品形态**：用户录入自己已发生的重大事件，系统自动做两件事：

1. **时辰校正**：如果用户不确定出生时辰，用事件反推最可能时辰。
2. **个人模型微调**：根据事件-预测匹配度，给不同 Agent 加权。

**录入事件类型**：升学、入职、跳槽、创业、结婚、分手、购房、生病、手术、获奖、搬家。

**反馈闭环**：

```
用户录入事件 → 系统对比各 Agent 对该时间段的预测 →
计算匹配度 → 调整 Agent 权重 / 提示时辰偏移 / 更新个人「命运基线」
```

**创新点**：让预测系统**越用越准**，而不是每次从零生成。

---

### 模块 4：择日行动引擎（Auspicious Action Engine）

**产品形态**：用户输入目标，系统返回「最佳行动窗口」并可一键加入日历。

**目标类型**：

- 求职面试、谈判签约、创业注册、产品发布
- 求婚、订婚、搬家、装修、手术
- 投资买入/卖出、出行、诉讼

**输出内容**：

- 推荐日期区间 + 每日评分
- 避开「冲克」时辰
- 与个人大运/流年匹配的解释
- 一键导出 `.ics` 到 Outlook / Google Calendar / 苹果日历

**创新点**：从「查黄历」升级为**个人目标优化求解器**。

---

### 模块 5：命运沙盒（Destiny Sandbox）

**产品形态**：A/B 对比界面，让用户探索「如果…会怎样」。

**沙盒场景**：

- **换时辰**：如果我是午时而不是巳时出生，命盘如何变化？
- **换出生地**：如果出生在北京而不是上海？
- **换选择**：如果我在 2024 年创业 vs 2026 年创业，哪个窗口更利？
- **换伴侣**：输入两人八字，对比「合婚指数」与潜在冲突领域。

**输出**：并排展示两个命盘的差异雷达图 + 关键领域对比 + Agent 辩论。

**创新点**：把命理从「宿命论」变成**决策模拟器**。

---

### 模块 6：命运剧本（Destiny Script）

**产品形态**：把命盘解释成一份「人生 RPG 角色卡」+「章节式剧本」。

**角色卡**：

- 命主属性（日主、格局、身强身弱）
- 天赋技能（用神领域）
- 弱点 debuff（忌神领域）
- 当前章节（大运主题）
- 下一章预告（未来大运关键词）

**章节剧本**：

- 把人生按大运分成「章节」，每个章节有主题、挑战、机遇、建议。
- 支持生成语音/视频版「命运解说」。

**创新点**：降低命理理解门槛，让年轻人愿意分享。

---

### 模块 7：隐私优先的联邦知识库

**产品形态**：后台 invisible feature。

**机制**：

- 用户命盘和事件默认**本地加密存储**（或私有云）。
- 只有经过去标识化、差分隐私处理后的「模式」才会进入全局知识库。
- 用户可贡献自己的标注案例，换取积分或高级功能。
- 全局知识库用于持续微调 Agent 的 domain 预测能力。

**创新点**：解决命理平台最大的信任问题——**我的命盘数据会不会被滥用**。

---

### 模块 8：命理 API 商店（MingMirror API Store）

**产品形态**：面向开发者。

**API 能力**：

- `/chart`：排盘（八字/紫微/七政）
- `/forecast`：指定时间窗口运势
- `/auspicious`：择日
- `/compatibility`：合婚
- `/report`：生成 PDF 报告
- `/agents/debate`：多 Agent 辩论

**创新点**：把命理能力产品化，让第三方 App / 小程序 / SaaS 可以集成。

## 6. 关键用户旅程

### 旅程 A：新用户首次使用

1. 打开首页 → 看到「生成你的命运数字孪生」CTA。
2. 输入出生信息（公历/农历、出生地、时辰）。
3. 选择「我想先了解」：事业 / 感情 / 健康 / 综合。
4. 系统生成 Life Twin Dashboard 与命运剧本首页。
5. 引导用户录入 1–3 件已发生大事，开启校准。
6. 推荐「命理议会」体验，展示多 Agent  debate。

### 旅程 B：决策场景

1. 用户目标：「我想在下半年跳槽」。
2. 进入择日行动引擎，输入目标。
3. 系统返回最佳面试/辞职/入职窗口。
4. 用户打开沙盒，对比「现在跳槽」vs「明年跳槽」。
5. 查看议会结论，保存决策备忘录。

### 旅程 C：命理师工作流

1. 命理师导入客户命盘。
2. 使用议会功能生成多角度分析。
3. 用报告模板生成 PDF 交付客户。
4. 客户反馈「准/不准」，数据回流校准模型。

## 8. 数据模型（核心）

### User
- id, phone/email, nickname, avatar
- privacy_level, created_at

### Chart
- id, user_id, type(bazi/ziwei/qizheng)
- birth_datetime, gender, location_json, timezone
- raw_pillars / raw_palaces（排盘结果）
- is_calibrated, confidence_score

### Event
- id, chart_id, event_type, happened_at, description
- predicted_by_agent（预测来源）
- match_score（预测与实际匹配度）

### Reading（分析记录）
- id, chart_id, question, strategy(single/council/debate)
- result_json, agent_weights, overall_confidence
- feedback_correct, feedback_note

### AuspiciousSlot
- id, chart_id, goal, start_date, end_date
- score, reasoning, recommended_actions
- exported_to_calendar

## 9. 技术架构

### 前端
- React 18 + TypeScript + Vite
- React Router 6
- Zustand（全局 UI 状态）+ TanStack Query（服务端状态）
- Tailwind CSS + shadcn/ui 组件库
- SVG/Canvas 自研排盘组件（核心资产）
- ECharts 趋势图
- Web Speech API / 视频生成服务做语音解读

### 后端
- FastAPI（已有）
- PostgreSQL + Redis
- 排盘引擎：Python 自研 + `skyfield`/`pyswisseph`
- Agent 编排：LangChain / 自研 multi-agent graph
- 任务队列：Celery + Redis
- 文件存储：S3/OSS
- 日历导出：icalendar

### AI / ML
- LLM：DeepSeek / GPT-4 / Claude（多模型 fallback）
- Embedding：bge-small-zh 做案例召回
- 微调：LoRA 在个人校准数据上微调轻量模型
- RLHF：根据用户反馈训练 reward model

### 隐私
- 本地优先存储（IndexedDB / SQLite WASM）
- 端到端加密敏感字段
- 差分隐私聚合后上传模式

## 12. 风险与合规

- **迷信/诈骗风险**：明确免责声明，不提供医疗/法律/投资建议，不卖改运产品。
- **数据隐私**：最小化收集，本地优先，符合《个人信息保护法》。
- **时辰准确性**：提示用户出生时间可能存在误差，提供校正工具但不保证 100% 准确。
- **模型幻觉**：议会机制 + 置信度 + caveat 降低单一模型胡说风险。
